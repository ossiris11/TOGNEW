# SEO-оптимизация TOGOSHOL

Документ для подготовки сайта к нормальной индексации после запуска домена.

## Цели

- Получать органический трафик по запросам про игровые ПК в Великом Новгороде.
- Сделать страницу понятной для поисковиков: кто продает, где работает, что предлагает, как связаться.
- Не ломать лендинг маркетинговым шумом: SEO-текст должен помогать клиенту, а не превращать сайт в простыню.

## Базовые запросы

- игровые ПК Великий Новгород;
- купить игровой компьютер Великий Новгород;
- сборка ПК на заказ Великий Новгород;
- компьютер под заказ Великий Новгород;
- игровой ПК RTX Ryzen;
- готовые сборки ПК;
- апгрейд игрового ПК;
- ПК для игр Full HD / 2K / 4K.

## Title и Description

Рекомендуемый `title`:

```text
TOGOSHOL - игровые ПК и сборка компьютеров на заказ в Великом Новгороде
```

Рекомендуемый `description`:

```text
Готовые игровые ПК и кастомные сборки TOGOSHOL в Великом Новгороде. Подбор комплектующих, стресс-тесты, настройка системы и помощь с выбором под бюджет.
```

## Контент, который стоит добавить

### Локальность

Добавить в текстовые блоки аккуратные упоминания:

- Великий Новгород;
- консультация по наличию;
- выдача/доставка по договоренности;
- связь через VK, Telegram и Max.

### Каталог

Для карточек товаров желательно хранить:

- уникальное название;
- цену;
- CPU;
- GPU;
- RAM;
- SSD;
- назначение: Full HD, 2K, 4K, работа;
- короткое описание без копипаста.

## Техническое SEO

Обязательный минимум:

- корректный `title` и `meta description`;
- Open Graph теги для VK/Telegram;
- фавиконка;
- `robots.txt`;
- `sitemap.xml`;
- canonical URL после выбора основного домена;
- нормальные `alt` у изображений товаров;
- человекочитаемые URL для будущих страниц товаров, если каталог станет многостраничным;
- сжатие изображений до WebP/AVIF там, где возможно;
- lazy loading для изображений ниже первого экрана;
- HTTPS без смешанного контента.

## Schema.org

После запуска домена стоит добавить JSON-LD:

- `LocalBusiness` или `ComputerStore`;
- `Product` для карточек ПК;
- `FAQPage` для блока FAQ;
- `AggregateRating` только если есть реальные подтвержденные отзывы.

Фейковые рейтинги не добавлять: это риск для доверия и поисковой выдачи.

## Индексация

После production-запуска:

1. Проверить, что сайт открывается по `https://домен`.
2. Проверить редирект с `http` на `https`.
3. Проверить отсутствие дублей `www`/без `www`.
4. Добавить сайт в Яндекс.Вебмастер.
5. Добавить сайт в Google Search Console.
6. Отправить `sitemap.xml`.
7. Проверить сниппет главной страницы.

## Метрики

Для оценки SEO отдельно смотреть:

- переходы на сайт;
- клики по контактам;
- заявки;
- популярные страницы/секции;
- конверсию из перехода в контакт;
- поисковые запросы в Яндекс.Вебмастере и Google Search Console.

Dashboard админки уже показывает базовые переходы, клики, заявки, топ страниц и состояние сервера.

## Что не делать

- Не писать длинные SEO-простыни в верхнем блоке.
- Не добавлять несуществующие отзывы и рейтинги.
- Не скрывать текст только для поисковиков.
- Не плодить одинаковые карточки товаров с разной ценой.
- Не коммитить реальные токены аналитики и доступы.
