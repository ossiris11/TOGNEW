# TOGOSHOL — Tasks for Codex

## Основная задача

Сделать frontend-часть продающего сайта **TOGOSHOL** для игровых ПК в Великом Новгороде.

Цель — не маркетплейс и не федеральный интернет-магазин, а нормальная локальная витрина с каталогом, подбором, условиями и быстрым переходом в диалог.

---

# Шаг 1. Проверить проект

Сначала изучить структуру проекта.

Если проект уже создан:

- не ломать текущую архитектуру;
- использовать существующий стек;
- сохранить рабочий импорт товаров, если он уже есть;
- не удалять реальные данные без причины.

Если проект пустой:

- создать React + TypeScript + Vite;
- настроить базовые стили;
- сделать компонентную структуру.

---

# Шаг 2. Зафиксировать каналы связи

Создать единый источник контактов:

```text
VK
Telegram
Max
Телефон
Email
```

Требования:

- не показывать пустые соцсети;
- не использовать фейковый телефон;
- все CTA должны вести в рабочий канал;
- VK Market не должен быть основным сценарием покупки.

---

# Шаг 3. Нормализовать данные товаров

Если товары импортируются из VK или другого источника, добавить слой нормализации.

Карточка должна получать:

```text
id
title
price
priceText
status
image
cpu
gpu
ram
storage
psu
cooling
class
scenario
sourceId
```

Убрать из карточек:

- рекламные хвосты;
- эмодзи-строки;
- “Конфигурация:”;
- грязные обрывки описаний;
- внешнюю market-ссылку как основной CTA.

---

# Шаг 4. Создать структуру компонентов

Минимальная структура:

```text
Header
Hero
FeaturedBuilds
ProductCatalog
ProductFilters
ProductCard
Configurator
CustomBuild
WhyTogoshol
TrustConditions
OrderProcess
Reviews
Faq
FinalCta
Footer
```

Данные вынести в массивы:

```text
products.ts
benefits.ts
conditions.ts
steps.ts
reviews.ts
faq.ts
```

---

# Шаг 5. Сделать дизайн-систему

Добавить CSS variables:

```css
--bg-main: #050505;
--bg-secondary: #08090B;
--bg-card: #101216;
--bg-card-soft: #15181D;
--text-main: #FFFFFF;
--text-muted: #A5A8AF;
--text-soft: #777B84;
--cyan: #00E5FF;
--blue: #1F6BFF;
--violet: #6A35FF;
--green: #4DFF88;
--border-soft: rgba(255,255,255,0.08);
--border-active: rgba(255,255,255,0.16);
--ease-premium: cubic-bezier(0.16, 1, 0.3, 1);
```

Тёмный premium-tech стиль оставить, но не превращать сайт в одну декоративную glow-картинку. Коммерческая ясность важнее эффекта.

---

# Шаг 6. Сделать Hero

Hero должен быть главным экраном.

Обязательно:

- H1: “Игровые ПК в Великом Новгороде”;
- subtitle;
- 2 CTA;
- 3 метрики;
- визуал ПК справа;
- CTA “Подобрать ПК” ведёт в конфигуратор;
- CTA “Смотреть сборки” ведёт в каталог;
- parallax visual на desktop;
- fade-up появление текста.

Если изображений нет, использовать CSS/SVG mockup игрового ПК.

---

# Шаг 7. Сделать Featured Builds

Показать 3-6 рекомендуемых сборок.

В каждой карточке:

- фото или fallback visual;
- status badge;
- title;
- цена;
- CPU / GPU;
- RAM / SSD;
- CTA “Написать по сборке”.

Не показывать все товары в hero.

---

# Шаг 8. Сделать Product Catalog

Добавить полноценную секцию “Все сборки”.

Обязательно:

- поиск;
- фильтр бюджета;
- фильтр класса сборки;
- сортировка;
- сетка карточек;
- “Показать ещё”;
- empty state.

Сетка:

- desktop: 3-4 карточки;
- tablet: 2 карточки;
- mobile: 1 карточка.

---

# Шаг 9. Сделать Product Card

Карточка должна быстро отвечать на вопросы:

- что за ПК;
- сколько стоит;
- какая видеокарта;
- какой процессор;
- сколько RAM;
- какой SSD;
- в наличии или под заказ;
- куда нажать, чтобы купить / обсудить.

CTA:

```text
Написать по сборке
Подобрать похожую
```

---

# Шаг 10. Сделать Configurator

Поля:

- бюджет;
- игры / задачи;
- разрешение;
- состояние комплектующих;
- RAM;
- SSD;
- стиль корпуса.

Результат:

- класс сборки;
- объяснение;
- предупреждение для нереалистичного бюджета;
- 2-3 ближайших товара;
- кнопка “Скопировать конфигурацию”;
- CTA “Отправить подбор”.

---

# Шаг 11. Сделать Custom Build

Одна широкая секция:

- “Соберём ПК под твою задачу”;
- короткий текст;
- 4 пункта;
- CTA;
- visual справа.

---

# Шаг 12. Сделать Why TOGOSHOL

4 карточки:

- Локально в Великом Новгороде;
- Подбор без лишней переплаты;
- Тестируем перед выдачей;
- Готово к запуску.

---

# Шаг 13. Сделать Trust Conditions

Добавить блок условий:

- гарантия;
- сроки;
- оплата;
- выдача / доставка.

Это обязательный блок. Без него сайт выглядит как прототип.

---

# Шаг 14. Сделать Order Process

Timeline из 5 шагов:

- Заявка;
- Подбор;
- Согласование;
- Сборка и тест;
- Передача.

Desktop — горизонтально или компактной сеткой.

Mobile — вертикально.

---

# Шаг 15. Сделать Reviews / Social Proof

Если есть реальные отзывы:

- показать 3-6 отзывов;
- указать источник;
- добавить ссылку на VK / соцсеть.

Если отзывов нет:

- скрыть секцию;
- не использовать фейковые отзывы.

Можно добавить галерею готовых работ, если есть реальные фото.

---

# Шаг 16. Сделать FAQ

Добавить вопросы:

- Можно ли собрать ПК под мой бюджет?
- Есть ли гарантия?
- Можно ли изменить готовую сборку?
- Сколько занимает сборка?
- Поможете ли с настройкой?

---

# Шаг 17. Сделать Final CTA и Footer

Final CTA:

- большой заголовок;
- короткий subtitle;
- VK / Telegram / phone buttons только если контакты реальные.

Footer:

- TOGOSHOL;
- описание;
- контакты;
- разделы;
- copyright.

---

# Шаг 18. Реализовать анимации

Обязательно:

1. Reveal on scroll.
2. Header blur on scroll.
3. Hero parallax.
4. RGB pulse только на декоративных элементах.
5. Card hover lift.
6. Button hover scale.
7. Mobile menu animation.
8. prefers-reduced-motion.

Анимации не должны мешать чтению каталога.

---

# Шаг 19. Адаптивность

Проверить:

- desktop 1440px;
- laptop 1280px;
- tablet 768px;
- mobile 430px;
- mobile 390px;
- mobile 360px.

На mobile:

- одна колонка;
- H1 не вылезает;
- кнопки удобно нажимать;
- карточки каталога в одну колонку;
- фильтры не ломают ширину;
- timeline вертикальный;
- нет горизонтального скролла.

---

# Шаг 20. Финальный polish

Дожать:

- отступы;
- typography scale;
- hover;
- glow;
- mobile spacing;
- контраст текста;
- кликабельность CTA;
- пустые состояния;
- реальные контакты;
- читаемость карточек.

---

# Запреты

Не делать:

- онлайн-оплату;
- корзину;
- регистрацию;
- личный кабинет;
- фейковые отзывы;
- фейковые телефоны;
- кнопки на пустые соцсети;
- длинные SEO-тексты;
- мегаменю уровня HYPERPC;
- внешний VK Market как главный путь покупки;
- визуальный шум вместо коммерческой ясности.

---

# Финальный критерий

Готовая страница должна ощущаться как нормальный локальный сайт по продаже игровых ПК: премиальный, понятный, с реальными сборками, условиями, подбором и быстрым переходом в диалог.
