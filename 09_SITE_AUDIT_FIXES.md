- нет нормального сценария выбора ПК;
- нет доверия: нет адреса, условий, гарантии, сроков, реальных контактов, соцдоказательств.

# P0 — критично исправить в первую очередь

## 1. В шапке стоят заглушки контактов

Сейчас:

- телефон: `+7 000 000-00-00`;
- Telegram: `https://t.me/`;
- email частично заглушечный;
- кнопка “Написать” ведёт не к реальному контакту.

Почему плохо:

- пользователь не сможет купить;
- доверие падает сразу;
- сайт выглядит недоделанным.

Исправление:

- заменить телефон на реальный;
- заменить Telegram на реальный `vk.me/tog_pc` или Telegram, если он есть;
- кнопку “Написать” вести в основной канал продаж;
- в footer продублировать реальные контакты.

## 2. Товары из VK импортированы, но данные грязные

Сейчас в карточки попадает текст вида:

- “💬 Напиши прямо сейчас получи подарок к ПК!”;
- “Конфигурация:”;
- обрывки описаний;
- строки с рекламными фразами вместо характеристик;
- местами опечатки из VK, например “Игрвоой”.

Почему плохо:

- карточки выглядят неаккуратно;
- характеристики плохо сканируются;
- премиальность ломается;
- пользователь не видит быстро CPU / GPU / RAM / SSD / цену.

Исправление:

- добавить слой нормализации данных после импорта;
- отделить маркетинговое описание от характеристик;
- карточка должна показывать только:
  - CPU;
  - GPU;
  - RAM;
  - SSD;
  - PSU / cooling, если есть;
  - цену;
  - статус.
- рекламные фразы вынести в маленький badge или убрать.

## 3. Товарная лента слишком низко и выглядит обрезанной

Сейчас:

- карточки начинаются почти под fold;
- на первом скрине видны не полностью;
- горизонтальная прокрутка ощущается как случайная полоска;
- часть карточек уходит вправо без явного сигнала, что это карусель.

Почему плохо:

- пользователь может не понять, что товаров много;
- главный коммерческий блок выглядит второстепенно;
- карточки не продают с первого взгляда.

Исправление:

- сделать ленту отдельной частью hero с заголовком “В наличии и под заказ”;
- добавить короткий счётчик: “26 сборок из VK”;
- добавить стрелки влево/вправо;
- добавить snap внутри горизонтальной ленты;
- активную карточку центрировать при клике;
- на desktop показывать 3–4 карточки полностью, а не обрезанные края.

## 4. Карточки товаров визуально слабые

Сейчас:

- карточки тёмные, но похожи на одинаковые прямоугольники;
- фото слишком маленькое и уходит вниз;
- у неактивных карточек мало информации;
- у активной карточки нет полноценного раскрытого состояния;
- характеристики появляются, но занимают мало места;
- CTA виден только в активной карточке.

Почему плохо:

- товар не ощущается “дорогим”;
- фото из VK не раскрывает ценность;
- пользователь вынужден кликать вслепую.

Исправление:

- сделать карточку в 2 слоя:
  - сверху фото ПК;
  - снизу название, цена, ключевые specs.
- для активной карточки:
  - фото крупнее;
  - specs в две колонки;
  - отдельные chips: `CPU`, `GPU`, `RAM`, `SSD`;
  - CTA “Написать по этой сборке”.
  - быстрые контакты: VK, Telegram, Max.
- для неактивной карточки показывать минимум:
  - фото;
  - GPU/CPU;
  - цену.

## 5. Нет полноценной секции каталога

Сейчас все товары спрятаны в горизонтальной ленте на hero.

Почему плохо:

- 26 товаров плохо просматривать только горизонтальным скроллом;
- нельзя сравнить;
- нет сортировки;
- нет быстрого поиска по бюджету.

Исправление:

- после hero добавить отдельную секцию “Все сборки”;
- сетка desktop: 3–4 карточки в ряд;
- tablet: 2 карточки;
- mobile: 1 карточка;
- добавить фильтры:
  - `до 60 000`;
  - `60 000–90 000`;
  - `90 000–150 000`;
  - `150 000+`;
  - `RTX 4060 / 4070 / 5080`;
  - `Intel / Ryzen`.

## 6. Scroll-snap сейчас мешает

Сейчас сайт прокручивается с привязкой к секциям.

Почему плохо:

- секции пустые, snap подчёркивает пустоту;
- пользователь теряет контроль;
- на тачпаде может быть неприятно;
- если контент секции больше экрана, часть может быть неудобно читать.

Исправление:

- отключить `scroll-snap-type` для обычного лендинга;
- если нужен слайдовый эффект, оставить только для desktop и только после визуального уплотнения секций;
- на mobile полностью отключить;
- заменить на плавные anchor transitions и reveal.

## 7. Второй, третий и четвёртый экраны слишком пустые

Сейчас:

- Custom Build занимает маленькую карточку в огромном пустом поле;
- Why TOGOSHOL выглядит как четыре карточки в пустоте;
- Order Process тоже висит в пустоте;
- Final CTA пустоват.

Почему плохо:

- сайт кажется недособранным;
- дорогое позиционирование не поддержано визуально;
- пользователь скроллит воздух.

Исправление:

- уменьшить высоту секций;
- убрать обязательный `min-height: 100svh` для всех `.section`;
- сделать секции естественной высоты;
- объединить некоторые блоки:
  - “Почему TOGOSHOL” + “Как проходит заказ” можно сделать одним экраном;
  - Custom Build можно расширить реальной формой подбора.

## 8. Общий фон всё ещё недостаточно управляемый

Сейчас:

- фон единый, но местами создаёт большие пустые тёмные поля;
- градиент не поддерживает композицию каждого блока;

  - `body::before` — базовый gradient;
  - `body::after` — мягкая сетка/световая маска с opacity 0.05–0.08;
  - локальные glow только внутри секций.
- не использовать резкие секционные background-переходы;
- добавить faint radial glow за товарной лентой.

Почему плохо:

H1 нормальный, но после импорта VK нужно точнее:

# Аудит модуля конфигуратора ПК

Файл модуля: `src/components/CustomBuild/CustomBuild.tsx`  
Стили: `src/components/CustomBuild/CustomBuild.css`  
Состояние: модуль рабочий, интерактивный, но пока больше похож на красивый калькулятор-демо, чем на точный коммерческий подбор.

## Что в модуле уже хорошо

- Есть понятная идея: пользователь задаёт бюджет, игру и разрешение.
- Слайдер бюджета работает.
- Сегментированные кнопки игры и разрешения работают.
- Правая карточка пересчитывается без перезагрузки.
- Есть рекомендованный класс сборки: START / PRO / ULTRA.
- Есть комплектующие: GPU, CPU, RAM, SSD, охлаждение.
- Есть оценка FPS.
- Есть CTA на обсуждение сборки.
- Визуально блок ближе к premium-tech, чем старый статичный custom-блок.
- На desktop структура понятная: слева управление, справа результат.

## Главная проблема

Конфигуратор сейчас считает условную рекомендацию сам по себе, а не опирается на реальные товары TOGOSHOL из VK.

Из-за этого может получиться конфликт:

- в каталоге есть реальные сборки за конкретную цену;
- конфигуратор рекомендует условный RTX 4070 Super / RTX 5080;
- пользователь нажимает CTA, а такой конкретной сборки может не быть в наличии.

Исправление:

- использовать `vkProducts` как источник реальных вариантов;
- подбирать ближайшие товары по бюджету;
- показывать “лучшие совпадения из наличия”;
- если точного совпадения нет, писать “под заказ”.

## Проблемы логики расчёта

### 1. FPS считается слишком грубо

Сейчас формула:

- берёт базовый FPS по игре;
- умножает на бюджет;
- умножает на коэффициент разрешения.

Почему плохо:

- бюджет не равен производительности;
- RTX 3060 Ti и RTX 5060 могут вести себя по-разному;
- CPU важен для CS2;
- 4K зависит почти полностью от GPU;
- “Работа / 3D / AI” вообще не должна показывать FPS как главный показатель.

Исправление:

- сделать таблицу GPU → performance score;
- сделать таблицу game/resolution → fps multiplier;
- для `Работа / 3D / AI` вместо FPS показывать:
  - “уровень рендера”;
  - “память для проектов”;
  - “AI/3D запас”.

### 2. Выбор 4K всегда включает ULTRA

Сейчас любое 4K автоматически даёт `Флагманский ULTRA`.

Почему плохо:

- пользователь с бюджетом 70 000 ₽ может выбрать 4K и увидеть RTX 5080;
- это создаёт нереалистичное ожидание.

Исправление:

- если бюджет низкий и выбран 4K, показывать warning:
  - “Для 4K комфортный бюджет обычно выше 160 000 ₽.”
- класс должен учитывать и бюджет, и разрешение, но не игнорировать ограничение бюджета.

### 3. Конфигуратор не показывает компромиссы

Сейчас он всегда уверенно рекомендует сборку.

Исправление:

# Итог

Главная проблема сайта сейчас не в том, что “всё плохо”. Проблема в том, что он застрял между двумя форматами:

- премиальный лендинг;
- сырой импорт VK-товаров.

Нужно разделить роли:

- hero продаёт бренд и быстро ведёт к подбору;
- каталог показывает все реальные товары;
- кастомный блок объясняет сборку под заказ;
- trust-блок снимает страхи;
- CTA ведёт в реальный канал продаж.

После этих правок сайт будет выглядеть не как прототип, а как нормальная локальная витрина игровых ПК.
